## 🔍 Project Overview

This repository contains the full outputs of the AmnestyDAO / Human Rights Token (HRT) Catalyst project, including research, tokenomics modelling, and a working MVP DAO application.

The project explores how decentralised governance systems can be applied within a global human rights movement, enabling supporters to:

- Create and vote on proposals  
- Receive and allocate tokens based on participation  
- Engage in transparent, participatory governance processes  

The MVP has been deployed and tested with a pilot group of Amnesty participants across multiple regions.

## ⚙️ MVP (Milestone 3)

The MVP DAO system includes:

- Proposal creation and management  
- Token-based voting  
- Token allocation mechanisms  
- Backend services for authentication and transaction handling  
- Frontend interface for user participation

---

## ⚠️ MVP Scope & Limitations

This MVP was developed for pilot testing and validation purposes.

- Identity is pseudonymous (wallet / login-based)  
- Advanced identity layers (DID, zk-ID) are not yet implemented  
- Security hardening (auth verification, audits, scaling) is planned for future phases  

The focus of this milestone is validating governance flows and user participation, rather than production-grade infrastructure.

## 🌍 Mission

To build a decentralized, ethical, and participatory system that:

- Democratises global human rights coordination  
- Incentivises verified advocacy through the Human Rights Token (HRT)  
- Protects dignity, memory, and participation through secure, privacy-first tools  
- Embeds equity, transparency, consent, and embodied governance into blockchain infrastructure  

---

## 📚 Key Research Insights

### Challenges Addressed

- Centralised legacy structures limit participation and plural representation  
- DAO ecosystems suffer from low voter turnout, plutocratic voting, and governance manipulation  
- Absence of consent-aware, culturally contextual, and ethical DAO frameworks  

### Strategic Design Elements

- Multi-tiered hybrid governance - DReps, Council of Stewards, Civic Assemblies  
- Quadratic and reputation-based voting to balance voice and influence  
- Privacy-first stack - zk-ID, Atala PRISM, Midnight integration, AES encryption  
- Embodied governance layer - cultural consent, dynamic visibility, guardian protocols

| HRT-GOV   | Soulbound    | Governance rights, proposal voting, reputation accrual         |

- Adaptive supply with participation-linked inflation logic  
- Burn-to-signal mechanisms to reinforce active and ethical governance  
- Reputation earned through advocacy, contribution, and peer validation

## ⚙️ Technical Stack (MVP Architecture)

- Identity - NuFi SSO, Atala PRISM, Midnight zk-ID  
- Governance - Clarity Protocol (CIP-1694-compliant), DReps, Council of Stewards  
- Smart Contracts - Aiken, Plutus, and zk-programs via Midnight  
- Storage - IPFS, Arweave, Filecoin for preserving memory, testimony, and protest metadata  
- Campaigns - MANUS AI assistant, QR protest check-ins (DePIN)  
- Metrics - SDG-aligned oracles and reputation-based dashboards  
- Interoperability - Bridges to EVM and Cosmos ecosystems
