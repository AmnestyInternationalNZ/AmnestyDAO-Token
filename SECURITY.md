## Security Policy
## Overview
The Human Rights DAO (HRDAO) project explores governance, participation, and
decentralised coordination systems for public-interest and human rights applications.
Security is essential to the integrity of the project. We encourage responsible disclosure
of vulnerabilities, privacy concerns, governance risks, and other issues that may affect
the safety, reliability, or trustworthiness of the platform.
This repository contains research, governance documentation, tokenomics models, pilot
materials, and software components developed during the HRDAO pilot.
## Supported Versions
At present, this repository is maintained as an open-source research and development
resource.
Security updates will be applied to actively maintained software components where
practical.

## Reporting Security Issues
If you discover a security vulnerability, privacy issue, governance exploit, or other
material risk, please report it responsibly.
Please do not disclose vulnerabilities publicly until the maintainers have had a
reasonable opportunity to investigate and respond.

## Responsible Disclosure
We request that researchers:
ï‚·Act in good faith.
ï‚·Avoid accessing, modifying, or deleting data beyond what is necessary to

ï‚·Avoid disrupting services or testing against live deployments without permission.
ï‚·Give maintainers reasonable time to investigate and remediate issues before

We will work collaboratively with researchers wherever possible.

Because this project explores technologies intended for human rights participation,

ï‚·Surveillance risks

## Security Philosophy
The HRDAO project follows several guiding principles:
ï‚·Open systems should be inspectable.
ï‚·Privacy should be preserved wherever practical.
ï‚·Security should be documented rather than obscured.

ï‚·Human rights considerations should inform technical decisions.
ï‚·Governance systems should be resilient to manipulation and capture.
Security is an ongoing process rather than a completed state.
